PIVMat Known software problems


If you find a bug not documented here, first have a look to the updated PIVMat Known software problems page online. If it is still undocumented, send an e-mail to moisy@fast.u-psud.fr.


  • When I use readimx or loadvec, I obtain the errors '??? Out of memory. Type HELP MEMORY for your options', or '??? ReadIMX error #4: Invalid data!'.
    This problem originates from a buggy release of LaVision's ReadIMX. Upgrade your ReadIMX package directly from the LaVision homepage (see the step 1 of the installation procedure).

  • When I use readimx or loadvec, I obtain the warning MEX-files with .dll extensions will not execute in a future version of MATLAB.
    Upgrade your version of readimx from the LaVision web site.

  • Compatibility issue. Since Matlab 7.4 (R2007a), the behavior of the command doc has changed: the command doc pivmat does not display the PIVMat toolbox start page in the help browser any more. If you use Matlab >= 7.4, type docpivmat to display the start page, or type docpivmat function_name to display the html documention of function_name in the help browser. If you use Matlab 7.3 or before, doc and docpivmat will have the same result.

  • When using file selection from a dialog box in loadvec, the titles from showvec are buggy.
